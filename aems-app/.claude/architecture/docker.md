# Docker

Docker Compose orchestration for the full Skeleton stack. Core services (client, server, database, redis) always run; everything else is gated behind **compose profiles**.

## Layout

- [docker-compose.yml](../../docker-compose.yml) ‚Äî root **shim** that `include:`s [docker/docker-compose.yml](../../docker/docker-compose.yml) with `project_directory: ./docker/`.
- [docker/docker-compose.yml](../../docker/docker-compose.yml) ‚Äî the real stack definition.
- [docker/database/](../../docker/database/) ‚Äî custom Postgres image with PostGIS installed ([Dockerfile](../../docker/database/Dockerfile), [postgis.sh](../../docker/database/postgis.sh)). Vanilla `postgres:16` will not work.
- [docker/proxy/](../../docker/proxy/) ‚Äî Traefik v3 dynamic config ([certs-traefik.yml](../../docker/proxy/certs-traefik.yml)). Terminates TLS, routes `/`, `/graphql`, `/api`, `/ext/*`, `/auth/sso/`.
- [docker/certs/](../../docker/certs/) ‚Äî cert provisioning helper ([Dockerfile](../../docker/certs/Dockerfile), [mkcert.sh](../../docker/certs/mkcert.sh)). Runs before proxy.
- [docker/keycloak/](../../docker/keycloak/) ‚Äî Keycloak image + [default-realm.json](../../docker/keycloak/default-realm.json) + [init-keycloak.sh](../../docker/keycloak/init-keycloak.sh).
- [docker/backup/](../../docker/backup/) ‚Äî backup sidecar (Node worker that snapshots Postgres on a schedule).
- [docker/map/](../../docker/map/) ‚Äî OSM tile server assets.
- [docker/wiki/](../../docker/wiki/) ‚Äî BookStack wiki service assets.
- [docker/seed/](../../docker/seed/) ‚Äî DB seed JSON.
- [docker/secrets/](../../docker/secrets/) ‚Äî host-side secret files (gitignored). Generated by `secrets.sh`/`secrets.ps1` at the repo root.

## Run from the repo root, not from `docker/`

The root [docker-compose.yml](../../docker-compose.yml) is a shim. **Always invoke `docker compose` from the repo root** so it auto-loads the root [.env](../../.env). Running compose from inside `docker/`, or with `-f docker/docker-compose.yml` from the root, makes compose look for `docker/.env` instead ‚Äî which is **not** where the project's env vars live.

```bash
# CORRECT ‚Äî picks up root .env and resolves build contexts correctly
docker compose up -d
docker compose --profile proxy --profile keycloak up -d

# WRONG ‚Äî loads docker/.env; wrong variables
cd docker && docker compose up -d
docker compose -f docker/docker-compose.yml up -d
```

## Compose profiles and service catalog

Core services run unprofiled; optional services attach profiles:

| Profile | Services | Image | Host ports | Internal DNS | Purpose |
|---|---|---|---|---|---|
| (none) | `client` | [docker/client/Dockerfile](../../docker/client/Dockerfile) | ‚Äî | `skeleton-client` | Next.js (prod build) |
| (none) | `server` | [docker/server/Dockerfile](../../docker/server/Dockerfile) | ‚Äî | `skeleton-server` | NestJS API |
| (none) | `database` | [docker/database/Dockerfile](../../docker/database/Dockerfile) | 5432 | `skeleton-database` | PostGIS Postgres 16 |
| (none) | `redis` | redis:7-alpine | 6379 | `skeleton-redis` | Pub/sub, sessions |
| `proxy` | `proxy` | Traefik v3 | 80, 443 | `skeleton-proxy` | TLS termination + routing |
| `keycloak` | `keycloak`, `keycloak-db` | [docker/keycloak/Dockerfile](../../docker/keycloak/Dockerfile) | ‚Äî | `skeleton-keycloak` | OAuth2 / SSO |
| `map` | `map` | [docker/map/Dockerfile](../../docker/map/Dockerfile) | ‚Äî | `skeleton-map` | MapLibre tile server |
| `nominatim` | `nominatim` | nominatim:4 | ‚Äî | `skeleton-nominatim` | Geocoding |
| `wiki` | `wiki`, `wiki-db` | BookStack + MariaDB | ‚Äî | `skeleton-wiki` | Internal wiki |

**Profile gating is load-bearing for minimal deploys** ‚Äî don't strip `profiles: [...]` to "make it simpler."

## Traefik routing

| Path pattern | Destination | Notes |
|---|---|---|
| `/` (all other) | `skeleton-client:3000` | Next.js app |
| `/graphql`, `/api/*`, `/ext/*` | `skeleton-server:3001` | NestJS API |
| `/auth/sso/*` | `skeleton-keycloak:8080` | Keycloak ‚Äî only with `keycloak` profile |

TLS terminates at Traefik. HTTP port 80 redirects to HTTPS port 443. mkcert certs are bind-mounted from `docker/certs/` (generated by `secrets.sh`).

## Secrets model

Two layers:

**Layer 1: Docker secrets** declared at the top of [docker/docker-compose.yml](../../docker/docker-compose.yml) ‚Äî session, JWT, DB passwords, Keycloak admin, BookStack, Nominatim. Each is a file in [docker/secrets/](../../docker/secrets/) mounted **read-only** into containers. **Files must exist before `docker compose up`** ‚Äî generate via the repo-root `secrets.sh` / `secrets.ps1`.

**Layer 2: Auto-generated at first boot** ‚Äî the backup sidecar's age-style encryption keypair. NOT a declared Docker secret (would require pre-existence); the [docker/backup/init-keys.sh](../../docker/backup/init-keys.sh) entrypoint writes into `./docker/secrets/backup/` mounted at `/host-secrets`.

[docker/secrets/](../../docker/secrets/) is gitignored. Don't commit anything there.

## Environment

- Root [.env](../../.env) ‚Äî non-secret defaults. Auto-loaded by compose only when invoked from the repo root.
- `docker/.env.secrets.docker` (if present) ‚Äî additional secret-style overrides.
- Compose interpolates `${VAR}` from the shell. [start-services.sh](../../start-services.sh) / [start-services.ps1](../../start-services.ps1) load `.env` before invoking compose.
- `COMPOSE_PROJECT_NAME` prefixes container names ‚Äî respect it when writing helper scripts.

## Networking

- Traefik terminates TLS at `:443` (and `:80` for redirect). Service containers do **not** expose host ports by default; they attach to the compose network and Traefik routes by Host/Path rule from container labels.
- The Traefik test dashboard is at `${TRAEFIK_TEST_PORT}` (when the `proxy` profile is active).
- `server.skeleton`, `client.skeleton`, etc. are internal DNS names within the compose network ‚Äî used in the `REWRITE_*_URL` vars to keep traffic inside the network.

## Build context

Image builds for `client`, `server`, `prisma`, and `common` use the **repo root as build context** (`context: ../`), so they can pull in sibling portal-linked modules. The Dockerfiles live in each sub-project. **Don't change the build context** without understanding the portal dependency chain.

## Operational reference

| Action | Command |
|---|---|
| First-time setup | `./secrets.sh` (or `.\secrets.ps1`), then `docker compose up -d` |
| Rebuild after code change | `docker compose build <service>` or `--build` on `up` |
| Reset DB (‚öÝ destructive) | `docker compose down -v` |
| Tail logs | `docker compose logs -f <service>` |
| Exec into DB | `docker compose exec database psql -U ...` |
| Restart single service | `docker compose restart skeleton-server` |

## Database image

[docker/database/Dockerfile](../../docker/database/Dockerfile) extends the official Postgres 16 image with PostGIS 3. **Never substitute `postgres:16`** ‚Äî the Prisma schema uses `Unsupported("geography(GEOMETRY,4326)")` columns that require PostGIS and would silently fail or error with the vanilla image.

## Gotchas

- **Secret files must exist before `up`** for any service that declares them. Ordering: `secrets.sh` ‚Üí `docker compose up`.
- **Running from `docker/`** ‚Üí Wrong `.env` loaded; auth / hostname vars are wrong. Always run from repo root.
- **DB image is custom** ‚Äî never replace with `postgres:16`; PostGIS will be missing.
- **Traefik v3 syntax** differs from v2. Check `traefik:v3.x` docs before copying older snippets.
- **Renaming a top-level secret** requires updating every `secrets: [...]` reference in each service.
- **The hostname in `.env` must be a real hostname** (not `localhost`) for OAuth + dev session sharing.
- **`docker compose down` (without `-v`) preserves volumes**; `down -v` wipes the DB.
- **Client `schema.graphql` not updating?** ‚Üí `autoSchemaFile` only writes in non-production. Force-regen: `yarn compile:schema` in the server workspace.
- **Local client can't connect to Docker API?** ‚Üí `REWRITE_*_URL` in `client/.env.local` must point to the real hostname, not `localhost`.
