Welcome to AEMS

Advanced Energy Management System

This is the default welcome page for AEMS deployments. To customize this page for your specific deployment, follow these steps:

Creating a Custom Deployment Template

  1. Create a new directory under /static/templates/ named {campus}-{building} using all lowercase letters (e.g., pnnl-rob, gmreit-2080oakley)
  2. Copy welcome.html and/or info.html from the default/ directory to your new directory
  3. Optionally create an images/ subdirectory for deployment-specific images
  4. Customize the HTML content as needed
  5. Set the environment variables in aems-app/.env:
    • VOLTTRON_CAMPUS - Your campus identifier (e.g., PNNL, GMREIT)
    • VOLTTRON_BUILDING - Your building identifier (e.g., ROB, 2080OAKLEY)

Template System Features

  • Pure HTML Fragments: Templates are HTML fragments that inherit all styling from the main application - no external CSS needed
  • Automatic Dark Mode: Templates automatically adapt to the user's light/dark theme preference
  • Environment-Driven: Automatically selects templates based on VOLTTRON_CAMPUS and VOLTTRON_BUILDING environment variables
  • Dev Preview Tool: Test templates at /dev/templates without changing environment variables
  • Easy Deployment: Simply copy template folders for new sites - no rebuild required for content changes

Directory Structure

/static/templates/
├── default/                      # Fallback templates
│   ├── images/                   # Default images
│   │   └── fig1.jpg ... fig15.jpg
│   ├── welcome.html
│   └── info.html
└── {campus}-{building}/          # Deployment-specific (lowercase)
    ├── images/                   # Optional deployment images
    │   └── building.jpg
    ├── welcome.html
    └── info.html (optional - falls back to default)

Important Notes

  • Image Paths: Always use absolute paths starting with /static/templates/ (e.g., /static/templates/default/images/fig1.jpg)
  • No CSS Links: Do not include <link> tags for CSS - styling is automatically inherited from the application
  • HTML Structure: Templates should be pure HTML fragments starting with a container div (no <!DOCTYPE>, <html>, <head>, or <body> tags)
  • Testing: Use the dev preview tool at /dev/templates to preview templates before deploying to production

Getting Started

The template system makes it easy to customize welcome and info pages for different AEMS deployments. Each deployment can have unique content and images while maintaining consistent styling across all sites. For detailed documentation, see the README at /static/templates/README.md.
